# Bibliomaniac — Concevoir, développer et déployer une application de bibliothèque numérique

**Assigné par :** Jean-Baptiste LAVISSE
**Créé le :** 16/04/2026
**Langue :** fr

Concevoir, prototyper, développer et déployer une application web de bibliothèque numérique permettant de rechercher des livres via l'API Google Books, consulter leurs informations et proposer une interface moderne, responsive et accessible.

## Référentiels

- [2023] Concepteur·rice développeur·se d'applications
- [2024] DevOps : appliquer les méthodes et outils de développement d'application

## Contexte du projet

Bibliomaniac souhaite proposer une application web intuitive permettant de rechercher des livres, consulter leurs informations et explorer facilement un catalogue sur mobile comme sur desktop.

Le projet combine :

- la conception UX/UI de l'application
- le prototypage des parcours principaux dans Figma
- l'intégration front-end avec React et Tailwind CSS
- l'ajout d'une recherche connectée à l'API Google Books
- l'organisation du travail en équipe avec GitHub
- la mise en ligne sur Vercel

L'application est pensée en **mobile first**, mais doit être responsive. Les maquettes mobiles servent de base de conception, sans être recopiées telles quelles. Les choix de navigation, de hiérarchie visuelle et de mise en page doivent être adaptés aux usages web.

## Modalités pédagogiques

### Phase 1 — Maquetter

L'équipe commence par concevoir les écrans clés de l'application dans Figma et prototyper les parcours principaux.

À produire :

- les écrans mobiles des vues principales
- un prototype cliquable

### Phase 2 — Organiser le projet

Le trinôme met ensuite en place un cadre de travail clair :

- initialiser le projet
- configurer Tailwind CSS, ESLint et Prettier
- créer le dépôt GitHub
- rédiger un README de démarrage
- créer un GitHub Project
- découper le travail en issues
- répartir clairement les premières tâches entre les trois membres

### Phase 3 — Développer l'interface

L'équipe développe l'application avec React et Tailwind CSS.

L'interface doit être :

- responsive
- accessible
- cohérente avec les maquettes
- structurée en composants réutilisables

**Point d'attention :** l'interface ne doit pas être une copie rigide de la maquette mobile. Certains choix purement mobiles devront être adaptés au web.

### Phase 4 — Ajouter la recherche de livres

L'application doit permettre :

- la recherche par titre, auteur ou ISBN
- l'affichage d'une liste de résultats
- l'affichage d'une fiche détaillée
- la gestion des états de chargement, d'erreur, d'absence de résultat et de données manquantes

### Phase 5 — Finaliser et déployer

Le trinôme doit :

- travailler avec une branche par fonctionnalité
- utiliser des pull requests et des commits explicites
- documenter le projet
- déployer l'application sur Vercel

Lefthook peut être ajouté en fin de projet pour automatiser certaines vérifications locales.
